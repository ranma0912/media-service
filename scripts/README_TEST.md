- 数据库测试
  - 数据库文件检查
  - 数据库大小检查
- 依赖测试
  - 后端依赖检查
  - 前端依赖检查
- 配置测试
  - 配置目录检查
  - 配置文件检查
- 文档测试

**文件名**: `test_statistics_api.py`

**功能**: 测试识别、整理和媒体库统计API

**测试内容**:
- 健康检查接口 (`/health`)
- 识别统计API (`/api/statistics/recognition`)

- 媒体库统计API (`/api/statistics/media`)
- 统计概览API (`/api/statistics/overview`)

## 使用方法

### 项目整体测试

#### 前置条件

1. **项目结构完整**
   - 确保所有必需的目录和文件存在
   - 检查项目配置文件

2. **后端服务运行**
   ```bash
   python -m app.main
   ```

3. **数据库已初始化**
   ```bash
   python -m app.main --init-db
   ```

#### 运行项目整体测试

```bash
cd scripts
python test_project.py
```

#### 指定项目根目录

```bash
python test_project.py c:\Users\ranma\CodeGeeXProjects\media-service
```

#### 指定输出文件

```bash
python test_project.py c:\Users\ranma\CodeGeeXProjects\media-service project_test_results.json
```

### 统计API测试

#### 前置条件

1. **启动后端服务**
   ```bash
   python -m app.main
   ```

2. **确保数据库已初始化**
   ```bash
   python -m app.main --init-db
   ```

3. **确保服务正常运行**
   - 访问 http://localhost:8000/health
   - 应返回: `{"status": "healthy", "version": "1.6.0"}`

### 运行测试脚本

#### 基本用法

```bash
cd scripts
python test_statistics_api.py
```

#### 指定服务地址

```bash
python test_statistics_api.py http://localhost:8000
```

#### 指定输出文件

```bash
python test_statistics_api.py http://localhost:8000 test_results.json
```

### 测试结果

测试脚本会输出以下信息：

1. **实时测试结果**
   - ✅ PASS - 测试通过
   - ❌ FAIL - 测试失败
   - 每个测试会显示详细的状态信息

2. **测试结果汇总**
   - 总测试数
   - 通过数
   - 失败数
   - 通过率

3. **测试结果文件**
   - 默认保存为 `test_results.json`
   - 包含所有测试的详细信息
   - 包含时间戳和测试摘要

## 故障排查

### 问题1: 服务未启动

**错误信息**:
```
❌ FAIL 健康检查
    错误: Connection refused
```

**解决方案**:
1. 检查后端服务是否启动
2. 检查服务地址和端口是否正确
3. 检查防火墙设置

### 问题2: 数据库未初始化

**错误信息**:
```
❌ FAIL 识别统计API
    错误: no such table: media_files
```

**解决方案**:
```bash
python -m app.main --init-db
```

### 问题3: 端口被占用

**错误信息**:
```
错误: [Errno 48] Address already in use
```

**解决方案**:
1. 查找占用端口的进程
   ```bash
   netstat -ano | findstr :8000
   ```
2. 终止进程或更改端口配置

## 注意事项

1. **测试环境**
   - 确保在测试环境中运行
   - 不要在生产环境中运行测试
   - 测试数据可能会被修改

2. **数据库备份**
   - 测试前建议备份数据库
   - 测试后可以恢复数据库

3. **日志查看**
   - 查看后端日志了解详细信息
   - 日志文件位置: `logs/app.log`

4. **清理测试数据**
   - 测试完成后可以清理测试数据
   - 或使用专门的测试数据库

## 扩展测试

如需添加更多测试用例，可以参考现有测试脚本的结构：

1. 继承 `StatisticsAPITester` 类
2. 添加新的测试方法
3. 在 `run_all_tests()` 中调用新方法
4. 确保使用 `print_test_result()` 记录结果
