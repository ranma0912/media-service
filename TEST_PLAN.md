
# MediaService 功能测试计划

## 测试环境准备

### 1. 环境要求
- Python 3.9+
- Node.js 16+
- Windows 10/11

### 2. 依赖安装

#### 后端依赖
```bash
pip install -r requirements.txt
```

#### 前端依赖
```bash
cd frontend
npm install
```

### 3. 数据库初始化
```bash
python -m app.main --init-db
```

### 4. 启动服务

#### 启动后端服务
```bash
python -m app.main
```

#### 启动前端开发服务器
```bash
cd frontend
npm run dev
```

## 测试用例

### 测试用例 1: 媒体文件列表加载

**前置条件:**
- 后端服务已启动
- 前端服务已启动
- 数据库中有媒体文件记录

**测试步骤:**
1. 打开浏览器，访问前端地址
2. 点击"媒体库"菜单
3. 观察媒体文件列表

**预期结果:**
- 媒体文件列表正常显示
- 文件信息完整（文件名、类型、大小等）
- 分页功能正常

### 测试用例 2: 创建扫描任务

**前置条件:**
- 后端服务已启动
- 前端服务已启动
- 已配置扫描路径

**测试步骤:**
1. 点击"扫描管理"菜单
2. 点击"立即扫描"按钮
3. 在确认对话框中点击"确定"

**预期结果:**
- 扫描任务成功创建
- 任务状态显示为"扫描中"
- 扫描任务列表中显示新任务

### 测试用例 3: 停止扫描任务

**前置条件:**
- 有正在运行的扫描任务

**测试步骤:**
1. 在扫描任务列表中找到运行中的任务
2. 点击"停止"按钮
3. 在确认对话框中点击"确定"

**预期结果:**
- 扫描任务成功停止
- 任务状态更新为"已停止"
- 停止成功提示显示

### 测试用例 4: 保存系统配置

**前置条件:**
- 后端服务已启动
- 前端服务已启动

**测试步骤:**
1. 点击"系统管理"菜单
2. 在"扫描配置"标签页修改配置
3. 点击"保存配置"按钮

**预期结果:**
- 配置成功保存
- 保存成功提示显示
- 配置值更新正确
