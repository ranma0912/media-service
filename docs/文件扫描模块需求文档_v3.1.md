## 一、概述

### 1.1 文档目的

本文档定义了文件扫描模块的完整功能需求，包括默认扫描策略、多路径配置、手动扫描、文件管理等功能。

### 1.2 适用范围

本文档适用于：
- 文件扫描模块的架构设计
- 前端扫描管理界面开发
- 后端扫描服务实现
- 测试用例设计

### 1.3 术语定义

| 术语         | 定义                                                   |
| ------------ | ------------------------------------------------------ |
| 默认扫描策略 | 用户配置的全局默认扫描策略，在未指定具体路径策略时生效 |
| 路径扫描策略 | 为特定扫描路径配置的独立扫描策略                       |
| 文件跳过策略 | 决定是否跳过某些文件的规则集合                         |
| 关键词库     | 包含特定关键词的规则库，用于识别需要跳过的文件         |
| 防抖时间     | 监控事件触发后的延迟等待时间，避免频繁触发             |
| 双重判定     | 通过文件扩展名和编码格式两者共同判定文件类型           |
| 文件哈希值   | 使用SHA-256算法计算的文件哈希值，用于唯一标识文件      |
| 文件队列     | 待扫描文件的处理队列，所有文件串行处理                 |
| 单文件批次   | 每个文件作为一个独立的批次，独立管理和执行             |

## 二、功能需求

### 2.1 默认扫描策略配置

#### 2.1.1 功能描述

支持用户建立默认文件扫描策略，默认文件扫描策略在用户仅指定扫描路径而不配置扫描策略时生效。

#### 2.1.2 功能详情

##### 2.1.2.1 默认扫描类型

**配置项**: `default_scan_type`

**可选值**:
- `full`: 全量扫描
- `incremental`: 增量扫描

**默认值**: `incremental`

**说明**: 
- 全量扫描：扫描所有符合条件的文件，不跳过任何文件
- 增量扫描：根据跳过策略，跳过部分文件

##### 2.1.2.2 默认递归方式

**配置项**: `default_recursive`

**可选值**:
- `true`: 递归扫描（扫描子目录）
- `false`: 非递归扫描（不扫描子目录）

**默认值**: `true`

**说明**: 决定是否递归扫描子目录

##### 2.1.2.3 默认文件跳过策略

**配置项**: `default_skip_strategy`

**可选值**:
- 当扫描类型为 `full` 时：固定为 `none`（不跳过任何文件）
- 当扫描类型为 `incremental` 时：
  - `keyword`: 仅跳过文件名命中关键词库词条的文件
  - `keyword_or_scanned`: 跳过文件名命中关键词库词条或已扫描的文件
  - `none`: 不跳过任何文件

**默认值**: `keyword`

**关键词库匹配规则**:
- 文件名包含关键词库中的任意词条即判定为命中
- 匹配不区分大小写
- 支持精确匹配和模糊匹配

**已扫描文件判定规则**:
- 文件扫描完成后生成SHA-256哈希值
- 判定文件是否已扫描时，比对当前文件与库中已扫描文件的哈希值
- 哈希值相同即判定为已扫描文件
- 哈希值不同则判定为新文件或已更新的文件
- 哈希值的计算在文件元数据提取完成后进行

##### 2.1.2.4 是否扫描子目录

**配置项**: `default_scan_subdirectories`

**可选值**:
- `true`: 扫描子目录
- `false`: 不扫描子目录

**默认值**: `true`

**说明**: 与递归方式功能相同，保留作为用户友好命名

##### 2.1.2.5 扫描任务监控防抖时间

**配置项**: `default_scan_debounce_time`

**取值范围**: 30-300秒

**默认值**: 30秒

**说明**: 
- 扫描进度更新的防抖时间
- 避免频繁更新进度影响性能
- 单位：秒

#### 2.1.3 存储方式

默认扫描策略存储在系统配置表中，配置键为 `scan.default_strategy`。

配置结构：
```json
{
  "scan_type": "incremental",
  "recursive": true,
  "skip_strategy": "keyword",
  "scan_subdirectories": true,
  "scan_debounce_time": 30
}
```

### 2.2 扫描路径配置

#### 2.2.1 功能描述

支持用户配置多个扫描路径，每个扫描路径支持配置独立的扫描策略。

#### 2.2.2 功能详情

##### 2.2.2.1 添加扫描路径

**操作流程**:
1. 用户点击"添加路径"按钮
2. 系统弹出路径配置对话框
3. 用户输入路径信息：
   - 路径名称（必填，用于识别和显示）
   - 扫描路径（必填，支持浏览选择）
   - 扫描策略配置
4. 系统验证路径有效性
5. 保存路径配置

**验证规则**:
- 路径名称：1-100字符，不能包含特殊字符
- 扫描路径：必须存在且为目录
- 同一路径不能重复添加

##### 2.2.2.2 扫描路径列表

**显示内容**:
- 路径名称
- 扫描路径
- 扫描类型
- 递归方式
- 文件跳过策略
- 是否扫描子目录
- 是否启用
- 是否开启文件监控
- 文件监控模式
- 文件监控防抖时间
- 扫描完成后是否自动识别
- 扫描完成后是否自动整理
- 最后扫描时间
- 扫描次数

**支持操作**:
- 启用/禁用路径
- 编辑路径配置
- 删除路径
- 立即扫描

##### 2.2.2.3 扫描策略配置

每个扫描路径支持以下策略参数：

**1. 扫描类型**
- `full`: 全量扫描
- `incremental`: 增量扫描

**2. 递归方式**
- `true`: 递归扫描
- `false`: 非递归扫描

**3. 文件跳过策略**
- `keyword`: 仅跳过关键词库文件
- `keyword_or_scanned`: 跳过关键词库和已扫描文件
- `none`: 不跳过任何文件

**4. 是否扫描子目录**
- `true`: 扫描子目录
- `false`: 不扫描子目录

**5. 扫描任务监控防抖时间**
- 取值范围：30-300秒
- 默认值：30秒

**6. 是否开启文件监控**
- `true`: 开启监控
- `false`: 关闭监控

**7. 文件监控模式**
- `watchdog`: 使用watchdog库监控文件变化
- `polling`: 使用轮询方式监控文件变化

**8. 文件监控防抖时间**
- 取值范围：1-60秒
- 默认值：5秒

**9. 扫描完成后是否自动识别**
- `true`: 自动识别
- `false`: 不自动识别

**10. 扫描完成后是否自动整理**
- `true`: 自动整理
- `false`: 不自动整理

#### 2.2.3 存储方式

扫描路径配置存储在 `scan_paths` 表中。

表结构：
```sql
CREATE TABLE scan_paths (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  path_name VARCHAR(100) NOT NULL,
  path VARCHAR(500) NOT NULL UNIQUE,
  
  -- 扫描策略
  scan_type VARCHAR(20) NOT NULL DEFAULT 'incremental',
  recursive BOOLEAN NOT NULL DEFAULT 1,
  skip_strategy VARCHAR(20) NOT NULL DEFAULT 'keyword',
  scan_subdirectories BOOLEAN NOT NULL DEFAULT 1,
  scan_debounce_time INTEGER NOT NULL DEFAULT 30,
  
  -- 监控配置
  monitoring_enabled BOOLEAN NOT NULL DEFAULT 1,
  monitoring_mode VARCHAR(20) NOT NULL DEFAULT 'watchdog',
  monitoring_debounce INTEGER NOT NULL DEFAULT 5,
  
  -- 自动化配置
  auto_recognize BOOLEAN NOT NULL DEFAULT 0,
  auto_organize BOOLEAN NOT NULL DEFAULT 0,
  
  -- 状态
  enabled BOOLEAN NOT NULL DEFAULT 1,
  
  -- 统计信息
  last_scan_at DATETIME,
  total_scans INTEGER DEFAULT 0,
  total_files_found INTEGER DEFAULT 0,
  
  -- 时间戳
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

### 2.3 手动触发扫描

#### 2.3.1 功能描述

支持用户手动触发扫描，包括扫描路径列表中的路径和临时指定的路径。

#### 2.3.2 扫描路径列表中的路径

**操作流程**:
1. 用户在路径列表中选中一个或多个路径
2. 用户点击"扫描"按钮
3. 系统弹出扫描选择对话框
4. 用户选择：
   - 使用默认扫描策略
   - 使用路径对应的扫描策略
5. 系统根据选择的策略创建扫描任务
6. 开始扫描并显示进度

**API接口**:
```
POST /api/scan/tasks
Body: {
  "path_ids": [1, 2, 3],  // 路径ID列表
  "use_default_strategy": false  // 是否使用默认策略
}
```

#### 2.3.3 临时指定的路径

**操作流程**:
1. 用户点击"扫描自定义路径"按钮
2. 系统弹出路径选择对话框
3. 用户选择扫描路径（支持浏览或手动输入）
4. 用户配置临时扫描策略：
   - 扫描类型（全量/增量）
   - 递归方式（递归/非递归）
   - 文件跳过策略
5. 系统验证路径有效性
6. 创建扫描任务
7. 开始扫描并显示进度

**API接口**:
```
POST /api/scan/tasks
Body: {
  "custom_path": "D:/Videos",  // 自定义路径
  "temp_strategy": {
    "scan_type": "full",
    "recursive": true,
    "skip_strategy": "none"
  }
}
```

### 2.4 文件队列管理

#### 2.4.1 功能描述

所有扫描文件按队列串行处理，不进行磁盘IO并发，避免磁盘IO风暴。

#### 2.4.2 队列管理规则

**单文件批次**:
- 每个文件作为一个独立的批次
- 每个文件有独立的任务ID和批次ID
- 批量操作选中N个文件即创建N个批次

**串行处理**:
- 所有文件按队列顺序串行处理
- 一次只处理一个文件
- 当前文件处理完成后，处理下一个文件
- 不进行磁盘IO并发

**文件队列结构**:
```
扫描队列
├─ 文件1 (批次1)
├─ 文件2 (批次2)
├─ 文件3 (批次3)
└─ 文件4 (批次4)
```

#### 2.4.3 队列管理实现

```python
class FileQueueManager:
    """文件队列管理器"""
    
    def __init__(self):
        self.file_queue = []  # 文件队列
        self.current_task = None  # 当前处理的任务
        self.is_processing = False  # 是否正在处理
    
    def add_file_task(self, file_path: str, strategy: dict) -> bool:
        """添加文件扫描任务"""
        # 1. 创建文件任务
        task_id = self._create_file_task(file_path, strategy)
        
        # 2. 添加到队列
        self.file_queue.append({
            'task_id': task_id,
            'file_path': file_path,
            'strategy': strategy
        })
        
        # 3. 尝试启动处理
        self._try_process_next()
        
        return True
    
    def _try_process_next(self):
        """尝试处理下一个文件"""
        # 检查是否正在处理
        if self.is_processing:
            return  # 正在处理，不启动新任务
        
        # 检查队列是否有任务
        if not self.file_queue:
            return  # 队列为空
        
        # 取出下一个文件
        task_config = self.file_queue.pop(0)
        
        # 标记为正在处理
        self.is_processing = True
        self.current_task = task_config['task_id']
        
        # 启动扫描
        self._start_scan(task_config)
    
    def _start_scan(self, task_config: dict):
        """启动文件扫描"""
        # 创建扫描器实例
        scanner = FileScanner(**task_config)
        
        # 执行扫描
        scanner.scan_file(task_config['file_path'])
    
    def on_task_completed(self, task_id: int):
        """任务完成回调"""
        # 检查是否是当前任务
        if self.current_task == task_id:
            # 重置处理状态
            self.is_processing = False
            self.current_task = None
            
            # 尝试处理下一个文件
            self._try_process_next()
```

### 2.5 文件列表显示

#### 2.5.1 功能描述

无论自动扫描还是手动扫描，在每个文件建立扫描任务后，均应在文件列表中显示文件信息。

#### 2.5.2 显示内容

**文件列表表格列**:
- 文件名
- 所在路径
- 扫描开始时间
- 扫描状态
- 扫描进度
- 扫描结束时间

**扫描状态定义**:
- `pending`: 等待中
- `scanning`: 扫描中
- `scanned`: 已扫描
- `recognizing`: 识别中
- `recognized`: 已识别
- `organizing`: 整理中
- `organized`: 已整理
- `completed`: 已完成
- `failed`: 失败
- `stopped`: 已停止

**扫描进度**:
- 0-100的百分比数值
- 实时更新

#### 2.5.3 文件列表操作

**单文件操作**:
1. 查看扫描结果
2. 重新扫描
3. 停止扫描
4. 删除扫描结果

**批量操作**:
1. 批量重新扫描
2. 批量停止扫描
3. 批量删除扫描结果

### 2.7 重新扫描

#### 2.7.1 功能描述

用户对选中文件执行重新扫描操作。

#### 2.7.2 操作流程

1. 用户在文件列表中选中一个或多个文件
2. 用户点击"重新扫描"按钮
3. 系统弹出确认对话框，显示：
   - 重新扫描文件数
   - 确认信息
4. 用户确认后：
   - 系统自动删除与文件相关的所有扫描记录
   - 重新建立扫描任务（每个文件一个独立批次）
5. 添加到文件队列
6. 开始扫描

#### 2.7.3 扫描策略选择

- 重新扫描时自动调用文件所在路径对应的扫描策略
- 若文件所在路径为手工扫描临时指定的，则调用默认扫描策略
- 无需用户选择

**API接口**:
```
POST /api/file-tasks/rescan
Body: {
  "task_ids": [1, 2, 3]
}
```

### 2.8 停止扫描

#### 2.8.1 功能描述

用户对选中文件执行停止扫描操作。

#### 2.8.2 操作流程

1. 用户在文件列表中选中一个或多个正在扫描的文件
2. 用户点击"停止扫描"按钮
3. 系统弹出确认对话框，显示：
   - 停止扫描文件数
   - 确认信息
4. 用户确认后：
   - 系统停止选中文件的实际扫描动作
   - 等待实际扫描动作停止
   - 自动删除与文件有关的所有扫描记录
   - 从文件队列中移除
   - 停止扫描任务

**API接口**:
```
POST /api/file-tasks/stop
Body: {
  "task_ids": [1, 2, 3]
}
```

### 2.9 删除扫描结果

#### 2.9.1 功能描述

用户对已完成扫描的文件执行删除扫描结果操作。

#### 2.9.2 操作限制

- 仅能对已完成扫描（状态为`scanned`、`completed`、`failed`）的文件执行删除操作
- 正在扫描（状态为`scanning`）的文件不能删除

#### 2.9.3 操作流程

1. 用户在文件列表中选中一个或多个已完成的文件
2. 用户点击"删除扫描结果"按钮
3. 系统弹出确认对话框，显示：
   - 删除文件数
   - 确认信息
4. 用户确认后：
   - 系统自动删除与文件有关的所有扫描记录
   - 删除文件扫描结果（MediaFile记录）

**API接口**:
```
DELETE /api/file-tasks
Body: {
  "task_ids": [1, 2, 3]
}
```

### 2.10 批量操作

#### 2.10.1 功能描述

支持用户对文件列表进行批量选中、批量重新扫描、批量停止扫描、批量删除扫描结果等操作。

#### 2.10.2 操作逻辑

批量操作逻辑与单文件操作一致，但弹出提示框应直接提示批量操作文件数。

**批量操作流程**:
1. 用户在文件列表中勾选多个文件
2. 用户点击对应的批量操作按钮
3. 系统弹出确认对话框，显示：
   - 批量操作文件数
   - 操作类型（重新扫描/停止扫描/删除结果）
   - 确认信息
4. 用户确认后执行操作
5. 每个文件创建独立的批次并添加到文件队列

**API接口**:
```
POST /api/file-tasks/batch/rescan
Body: {
  "task_ids": [1, 2, 3, 4, 5]
}

POST /api/file-tasks/batch/stop
Body: {
  "task_ids": [1, 2, 3, 4, 5]
}

DELETE /api/file-tasks/batch
Body: {
  "task_ids": [1, 2, 3, 4, 5]
}
```

## 三、文件类型支持

### 3.1 文件类型判定规则

文件类型通过**文件扩展名**和**文件编码格式**双重判定，必须同时符合才能判定为可扫描的文件类型。

**判定流程**:
1. 检查文件扩展名是否在支持列表中
2. 提取文件编码格式
3. 检查编码格式是否在支持列表中
4. 两者都符合才加入扫描队列

### 3.2 支持的文件扩展名

| 扩展名 | 类型 | 说明                                 |
| ------ | ---- | ------------------------------------ |
| .mp4   | 视频 | MPEG-4 Part 14                       |
| .mkv   | 视频 | Matroska Video Container             |
| .avi   | 视频 | Audio Video Interleave               |
| .mov   | 视频 | QuickTime File Format                |

### 3.4 文件哈希值计算

**哈希算法**: SHA-256

**计算时机**:
- 文件元数据提取完成后
- 文件扫描完成前

**存储位置**:
- media_files 表的 sha256_hash 字段

**哈希值用途**:
- 唯一标识文件内容
- 判定文件是否已扫描
- 检测文件是否被修改

**计算方式**:
```python
import hashlib

def calculate_file_hash(file_path: str) -> str:
    """
    计算文件的SHA-256哈希值
    
    Args:
        file_path: 文件路径
    
    Returns:
        SHA-256哈希值（十六进制字符串）
    """
    sha256_hash = hashlib.sha256()
    
    # 分块读取文件，避免内存溢出
    with open(file_path, 'rb') as f:
        # 每次读取8MB
        for chunk in iter(lambda: f.read(8192 * 1024), b''):
            sha256_hash.update(chunk)
    
    return sha256_hash.hexdigest()
```

### 3.5 已扫描文件判定机制

**判定流程**:
1. 扫描文件时，计算当前文件的SHA-256哈希值
2. 查询数据库中是否存在相同哈希值的文件
3. 根据扫描类型和跳过策略决定是否跳过：
   - skip_strategy = "keyword": 仅检查关键词库
   - skip_strategy = "keyword_or_scanned": 检查关键词库和哈希值
   - skip_strategy = "none": 不跳过任何文件

**判定逻辑**:
```python
def is_file_scanned(file_path: str, skip_strategy: str) -> bool:
    """
    判断文件是否已扫描
    
    Args:
        file_path: 文件路径
        skip_strategy: 跳过策略
    
    Returns:
        是否应跳过扫描
    """
    # 1. 计算当前文件的哈希值
    current_hash = calculate_file_hash(file_path)
    
    # 2. 检查关键词库
    if skip_strategy in ['keyword', 'keyword_or_scanned']:
        if is_keyword_matched(file_path):
            return True
    
    # 3. 检查是否已扫描（通过哈希值）
    if skip_strategy == 'keyword_or_scanned':
        existing_file = db.query_media_file_by_hash(current_hash)
        if existing_file:
            return True
    
    return False
```

def is_supported_video_file(file_path: Path) -> bool:
    """
    检查文件是否为支持的视频文件
    
    Args:
        file_path: 文件路径
    
    Returns:
        是否为支持的视频文件
    """
    # 1. 检查文件扩展名
    supported_extensions = {
        '.mp4', '.mkv', '.avi', '.mov', '.rmvb', '.ts', '.m2ts', '.mts',
        '.wmv', '.flv', '.webm', '.vob', '.ogv', '.ogg', '.mpg', '.mpeg',
        '.m4v', '.3gp', '.3g2', '.f4v', '.f4p'
    }
    
    if file_path.suffix.lower() not in supported_extensions:
        return False
    
    # 2. 提取编码格式
    try:
        media_info = MediaInfo.parse(str(file_path))
        video_track = next((t for t in media_info.tracks if t.track_type == 'Video'), None)
        
        if not video_track:
            return False
        
        codec_id = video_track.codec_id.lower() if video_track.codec_id else ''
        
        # 3. 检查编码格式
        supported_codecs = {

            'h264', 'h265', 'hevc', 'vp9'
        }
        
        # 标准化编码格式
        normalized_codec = codec_id.replace(' ', '').lower()
        
        return any(codec in normalized_codec for codec in supported_codecs)
        
    except Exception as e:
        logger.error(f"提取文件编码格式失败 {file_path}: {e}")
        return False

## 四、扫描线程管理

### 4.1 功能描述

所有扫描文件按队列串行处理，避免磁盘IO并发导致的磁盘IO风暴。

### 4.2 队列管理规则

**单文件批次**:
- 每个文件作为一个独立的批次
- 每个文件有独立的任务ID和批次ID
- 批量操作选中N个文件即创建N个批次

**串行处理**:
- 所有文件按队列顺序串行处理
- 一次只处理一个文件
- 当前文件处理完成后，处理下一个文件
- 不进行磁盘IO并发

**文件队列结构**:
```
文件队列
├─ 文件1 (批次1)
├─ 文件2 (批次2)
├─ 文件3 (批次3)
└─ 文件4 (批次4)
```

**执行规则**:
- 一次只执行1个文件的扫描任务
- 任务完成后从队列中取出下一个文件
- 按FIFO（先进先出）顺序处理

### 4.3 队列管理实现

```python
class FileQueueManager:
    """文件队列管理器"""
    
    def __init__(self):
        self.file_queue = []  # 文件队列
        self.current_task = None  # 当前处理的任务
        self.is_processing = False  # 是否正在处理
    
    def add_file_task(self, file_path: str, strategy: dict) -> bool:
        """添加文件扫描任务"""
        # 1. 创建文件任务
        task_id = self._create_file_task(file_path, strategy)
        
        # 2. 添加到队列
        self.file_queue.append({
            'task_id': task_id,
            'file_path': file_path,
            'strategy': strategy
        })
        
        # 3. 尝试启动处理
        self._try_process_next()
        
        return True
    
    def _try_process_next(self):
        """尝试处理下一个文件"""
        # 检查是否正在处理
        if self.is_processing:
            return  # 正在处理，不启动新任务
        
        # 检查队列是否有任务
        if not self.file_queue:
            return  # 队列为空
        
        # 取出下一个文件
        task_config = self.file_queue.pop(0)
        
        # 标记为正在处理
        self.is_processing = True
        self.current_task = task_config['task_id']
        
        # 启动扫描
        self._start_scan(task_config)
    
    def _start_scan(self, task_config: dict):
        """启动文件扫描"""
        # 创建扫描器实例
        scanner = FileScanner(**task_config)
        
        # 执行扫描
        scanner.scan_file(task_config['file_path'])
    
    def on_task_completed(self, task_id: int):
        """任务完成回调"""
        # 检查是否是当前任务
        if self.current_task == task_id:
            # 重置处理状态
            self.is_processing = False
            self.current_task = None
            
            # 尝试处理下一个文件
            self._try_process_next()
```

class ProgressReporter:
    """进度报告器"""
    
    def __init__(self, debounce_time: int = 30):
        self.debounce_time = debounce_time  # 防抖时间（秒）
        self.last_update_time = None
        self.pending_update = None
        self.update_timer = None
    
    async def report_progress(self, progress_data: dict, force: bool = False):
        """报告进度"""
        now = datetime.now()
        
        # 强制更新或防抖时间已过
        if force or not self.last_update_time or (now - self.last_update_time).total_seconds() >= self.debounce_time:
            await self._send_progress(progress_data)
            self.last_update_time = now
        else:
            # 保存待发送的进度
            self.pending_update = progress_data
            
            # 设置定时器
            if not self.update_timer:
                import asyncio
                self.update_timer = asyncio.create_task(self._debounced_send())
    
    async def _debounced_send(self):
        """防抖发送"""
        await asyncio.sleep(self.debounce_time)
        if self.pending_update:
            await self._send_progress(self.pending_update)
            self.last_update_time = datetime.now()
            self.pending_update = None
        self.update_timer = None
    
    async def _send_progress(self, progress_data: dict):
        """发送进度"""
        await manager.send_progress(progress_data['task_id'], progress_data)

## 九、非功能需求

### 9.1 性能需求

- 所有扫描文件按队列串行处理，避免磁盘IO并发
- 进度更新应使用防抖机制，避免频繁更新
- 大文件扫描应支持断点续传
- 数据库操作应使用索引优化查询性能
- SHA-256哈希值计算应使用分块读取，避免内存溢出
- 避免磁盘IO风暴：单线程处理所有文件

### 9.2 可靠性需求

- 扫描任务异常终止时应保存当前进度
- 网络断开时应支持重连和状态恢复
- 数据库操作失败时应记录错误日志
- 磁盘IO错误应友好提示用户
- 哈希值计算失败应记录错误日志并跳过文件

### 9.3 可扩展性需求

- 支持动态添加扫描路径
- 支持自定义文件类型判定规则
- 支持插件化扩展编码格式识别
- 支持自定义跳过策略规则
- 支持多种哈希算法（预留）

### 9.4 可维护性需求

- 代码应模块化，职责清晰
- 配置应集中管理
- 日志应详细记录关键操作
- 应提供完善的错误处理机制
- 哈希值计算应提供单独的工具函数，便于测试和维护

### 10.4 SHA-256哈希值说明

**算法特点**:
- SHA-256是安全哈希算法，生成256位（32字节）哈希值
- 输出为64个十六进制字符
- 对于相同文件，哈希值永远相同
- 对于不同文件，哈希值几乎不可能相同

**应用场景**:
- 文件去重：相同哈希值表示相同文件
- 文件变更检测：哈希值不同表示文件已修改
- 已扫描判定：通过哈希值判断文件是否已扫描

**计算优化**:
- 使用分块读取（每次8MB）避免内存溢出
- 适合大文件处理
- 计算速度约为200-400MB/s（取决于硬件）

**安全性**:
- SHA-256是密码学安全哈希函数
- 不可逆：无法从哈希值还原原始文件
- 抗碰撞性：很难找到两个不同文件产生相同哈希值

### 10.5 单文件批次说明

**设计原则**:
- 每个文件作为一个独立的批次
- 每个文件有独立的任务ID和批次ID
- 不再使用目录级别的批次管理

**优势**:
- 细粒度控制：每个文件独立管理
- 串行处理：避免磁盘IO并发
- 防止IO风暴：单线程处理所有文件
- 简化逻辑：不需要复杂的并发控制

**实现方式**:
- 文件队列：FIFO队列，所有文件按顺序处理
- 一次一文件：当前文件处理完成后，处理下一个
- 批量操作：选中N个文件即创建N个批次
