## 一、系统架构概述

### 核心组件

```
┌─────────────────────────────────────────────────────────────┐
│                      前端界面                            │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐ │
│  │ 扫描路径管理 │  │ 扫描任务管理 │  │ 文件任务管理 │ │
│  └──────────────┘  └──────────────┘  └──────────────┘ │
└────────────────────┬────────────────────────────────────┘
                     │ WebSocket / HTTP API
                     ↓
┌─────────────────────────────────────────────────────────────┐
│                      后端服务                            │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐ │
│  │ 扫描队列管理 │  │ 扫描器      │  │ 进度报告器  │ │
│  └──────────────┘  └──────────────┘  └──────────────┘ │
│         ↓                ↓                ↓              │
│  ┌──────────────────────────────────────────────┐       │
│  │              数据库层                       │       │
│  │  scan_paths | scan_history | file_tasks    │       │
│  │  media_files | keyword_rules              │       │
│  └──────────────────────────────────────────────┘       │
└─────────────────────────────────────────────────────────────┘
```

### 数据模型关系

```
scan_paths (扫描路径)
   ├─ id
   ├─ path_name
   ├─ path
   ├─ scan_type (扫描类型)
   ├─ recursive (递归方式)
   ├─ skip_strategy (跳过策略)
   ├─ scan_subdirectories (是否扫描子目录)
   ├─ scan_debounce_time (防抖时间)
   ├─ monitoring_enabled (是否监控)
   ├─ monitoring_mode (监控模式)
   ├─ monitoring_debounce (监控防抖)
   ├─ auto_recognize (自动识别)
   ├─ auto_organize (自动整理)
   └─ enabled (是否启用)

scan_history (扫描历史)
   ├─ id (任务ID)
   ├─ batch_id (批次ID)
   ├─ target_path (扫描路径)
   ├─ scan_type (扫描类型)
   ├─ recursive (递归方式)
   ├─ total_files (文件总数)
   ├─ new_files (新增文件)
   ├─ updated_files (更新文件)
   ├─ skipped_files (跳过文件)
   ├─ failed_files (失败文件)
   ├─ status (状态)
   ├─ started_at (开始时间)
   └─ completed_at (完成时间)

file_tasks (文件任务)
   ├─ id (文件任务ID)
   ├─ media_file_id (媒体文件ID)
   ├─ scan_batch_id (批次ID)
   ├─ file_name (文件名)
   ├─ file_path (文件路径)
   ├─ status (状态)
   ├─ scan_started_at (扫描开始)
   ├─ scan_completed_at (扫描完成)
   ├─ scan_progress (扫描进度)
   ├─ video_tracks (视频轨道)
   ├─ audio_tracks (音频轨道)
   ├─ subtitle_tracks (字幕轨道)
   ├─ video_codec (视频编码)
   ├─ has_external_subtitle (外挂字幕)
   ├─ external_subtitle_name (外挂字幕名)
   └─ scan_result (扫描结果)

media_files (媒体文件)
   ├─ id
   ├─ file_path
   ├─ file_name
   ├─ file_extension
   ├─ file_size
   ├─ sha256_hash (文件哈希值)
   ├─ video_codec
   ├─ audio_codec
   ├─ duration
   ├─ width
   ├─ height
   └─ media_type

keyword_rules (关键词规则)
   ├─ id
   ├─ library_id (关键词库ID)
   ├─ pattern (匹配模式)
   ├─ priority (优先级)
   └─ is_enabled (是否启用)
```

## 二、主要流程图

### 2.1 默认扫描策略配置流程

```
用户操作
   ↓
打开系统设置
   ↓
选择"默认扫描策略"
   ↓
系统加载当前策略配置
   ├─ 从配置表读取 scan.default_strategy
   └─ 显示策略参数
      ├─ 扫描类型: 全量/增量
      ├─ 递归方式: 递归/非递归
      ├─ 文件跳过策略: 根据扫描类型显示选项
      ├─ 是否扫描子目录: 是/否
      └─ 扫描任务监控防抖时间: 30-300秒
   ↓
用户修改策略参数
   ↓
【动态更新跳过策略选项】
   ↓
用户选择扫描类型
   ↓
系统动态更新跳过策略选项
   ├─ 如果选择"全量扫描"
   │  └─ 仅显示"不跳过任何文件"选项
   │
   └─ 如果选择"增量扫描"
      └─ 显示三个选项:
         ├─ 仅跳过关键词库文件
         ├─ 跳过关键词库或已扫描文件
         └─ 不跳过任何文件
   ↓
用户点击"保存"
   ↓
前端验证输入
   ├─ 防抖时间: 30-300秒范围验证
   ├─ 必填项检查
   └─ 数据类型验证
   ↓
调用API: PUT /api/scan/default-strategy
   ↓
后端保存策略
   ├─ 更新配置表
   ├─ 序列化为JSON
   └─ 记录配置历史
   ↓
返回保存结果
   ├─ 成功: 200 OK
   └─ 失败: 400 Bad Request
   ↓
前端显示结果
   ├─ 成功提示
   └─ 刷新策略显示
```

### 2.2 添加扫描路径流程

```
用户操作
   ↓
点击"添加路径"按钮
   ↓
打开路径配置对话框
   ↓
【路径基本信息】
   ↓
用户输入路径名称
   ├─ 必填，1-100字符
   └─ 不能包含特殊字符
   ↓
用户选择扫描路径
   ├─ 支持浏览按钮选择
   └─ 支持手动输入
   ↓
【扫描策略配置】
   ↓
配置扫描类型
   ├─ 全量扫描
   └─ 增量扫描
   ↓
配置递归方式
   ├─ 递归扫描
   └─ 非递归扫描
   ↓
配置文件跳过策略
   ├─ 仅跳过关键词库文件
   ├─ 跳过关键词库或已扫描文件
   └─ 不跳过任何文件
   ↓
配置是否扫描子目录
   ├─ 是
   └─ 否
   ↓
配置扫描任务监控防抖时间
   ├─ 默认: 30秒
   └─ 范围: 30-300秒
   ↓
【文件监控配置】
   ↓
配置是否开启文件监控
   ├─ 是
   └─ 否
   ↓
配置文件监控模式
   ├─ watchdog (实时监控)
   └─ polling (轮询监控)
   ↓
配置文件监控防抖时间
   ├─ 默认: 5秒
   └─ 范围: 1-60秒
   ↓
【自动化配置】
   ↓
配置扫描完成后是否自动识别
   ├─ 是
   └─ 否
   ↓
配置扫描完成后是否自动整理
   ├─ 是
   └─ 否
   ↓
用户点击"确定"
   ↓
前端验证输入
   ├─ 路径名称: 非空、长度、特殊字符
   ├─ 扫描路径: 非空、路径有效性
   ├─ 路径唯一性: 检查是否已存在
   └─ 防抖时间: 范围验证
   ↓
调用API: POST /api/scan/paths
   ↓
后端验证和处理
   ├─ 检查路径是否存在
   │  └─ 不存在 → 返回404
   ├─ 检查路径是否为目录
   │  └─ 不是目录 → 返回400
   ├─ 检查路径是否已配置
   │  └─ 已存在 → 返回409
   └─ 创建扫描路径记录
      ├─ 保存到 scan_paths 表
      ├─ 初始化统计字段
      │  ├─ total_scans = 0
      │  ├─ total_files_found = 0
      │  ├─ last_scan_at = null
      │  └─ last_scan_batch_id = null
      └─ 记录创建时间
   ↓
返回创建结果
   ├─ 成功: 返回路径ID和基本信息
   └─ 失败: 返回错误信息
   ↓
前端显示结果
   ├─ 成功提示
   └─ 刷新路径列表
```

### 2.3 手动触发扫描流程（路径列表）

```
用户操作
   ↓
在路径列表中选中一个或多个路径
   ↓
点击"扫描"按钮
   ↓
弹出扫描选择对话框
   ↓
显示选中路径信息
   ├─ 路径名称列表
   ├─ 扫描路径列表
   └─ 当前策略摘要
   ↓
用户选择扫描策略
   ├─ 使用默认扫描策略
   │  ├─ 读取全局默认策略
   │  └─ 应用到所有选中路径
   │
   └─ 使用路径对应的扫描策略
      ├─ 使用每个路径自己的策略
      └─ 策略可能不同
   ↓
用户确认选择
   ↓
前端构建扫描请求
   ├─ path_ids: [id1, id2, ...]
   └─ use_default_strategy: true/false
   ↓
调用API: POST /api/scan/tasks
   ↓
后端处理请求
   ├─ 验证路径ID有效性
   ├─ 读取路径配置
   ├─ 确定使用的策略
   │  └─ use_default_strategy = true
   │     ├─ 读取默认策略
   │     └─ 应用默认策略
   │
   │  └─ use_default_strategy = false
   │     └─ 使用路径自身策略
   │
   ├─ 识别磁盘
   │  ├─ 提取路径的磁盘盘符
   │  └─ Windows: C:, D:, E: ...
   │
   └─ 创建扫描任务
      ├─ 生成批次ID (batch_id)
      ├─ 创建 scan_history 记录
      ├─ 创建 scan_progress 记录
      └─ 提交到扫描队列
   ↓
【扫描队列管理】
   ↓
扫描队列管理器接收任务
   ├─ 识别磁盘盘符
   ├─ 检查磁盘队列
   │  ├─ 队列不存在 → 创建队列
   │  └─ 队列已存在 → 添加到队列
   ├─ 检查磁盘活跃任务
   │  └─ 无活跃任务 → 启动任务
   │     ├─ 从队列取出任务
   │     ├─ 创建扫描器实例
   │     └─ 启动后台扫描
   │
   └─ 有活跃任务 → 等待
   ↓
返回任务信息
   ├─ task_id: 任务ID
   ├─ batch_id: 批次ID
   ├─ status: "accepted"
   └─ message: "扫描任务已创建"
   ↓
前端显示任务
   ├─ 添加到扫描任务列表
   ├─ 连接WebSocket监听进度
   └─ 实时更新进度显示
```

### 2.5 文件类型双重判定流程

```
扫描器发现文件
   ↓
【第一步: 检查文件扩展名】
   ↓
提取文件扩展名
   ├─ 获取文件名后缀
   └─ 转换为小写
   ↓
检查扩展名是否在支持列表中
   ├─ 支持的扩展名:
   │  ├─ .mp4, .mkv, .avi, .mov
   │  ├─ .rmvb, .ts, .m2ts, .mts
   │  ├─ .wmv, .flv, .webm, .vob
   │  ├─ .ogv, .ogg, .mpg, .mpeg
   │  ├─ .m4v, .3gp, .3g2, .f4v, .f4p
   │  └─ ... (共22种)
   │
   ├─ 扩展名在列表中 → 继续
   └─ 扩展名不在列表中 → 跳过文件
      └─ 不加入扫描队列
   ↓
【第二步: 检查编码格式】
   ↓
使用 pymediainfo 提取媒体信息
   ├─ MediaInfo.parse(file_path)
   ├─ 获取视频轨道
   └─ 提取 codec_id
   ↓
检查是否包含视频轨道
   ├─ 有视频轨道 → 继续
   └─ 无视频轨道 → 跳过文件
   ↓
标准化编码格式
   ├─ 转换为小写
   ├─ 移除空格
   └─ 处理常见别名
   ↓
检查编码格式是否在支持列表中
   ├─ 支持的编码格式:
   │  ├─ mpeg-1, mpeg-2, mpeg-4
   │  ├─ x264, x265
   │  ├─ av1
   │  ├─ vc9
   │  ├─ h264, h265, hevc
   │  └─ vp9
   │
   ├─ 编码格式在列表中 → 加入扫描队列
   │  └─ 创建文件任务
   │     ├─ file_tasks 表
   │     ├─ status = "pending"
   │     └─ 关联 media_file
   │
   └─ 编码格式不在列表中 → 跳过文件
      └─ 不加入扫描队列
```

### 2.6 文件哈希值计算流程

```
文件通过类型判定
   ↓
准备计算哈希值
   ├─ 初始化 SHA-256 哈希对象
   └─ 创建哈希上下文
   ↓
【分块读取文件】
   ↓
打开文件（二进制模式）
   ↓
循环读取文件块
   ├─ 每次读取 8MB 数据
   ├─ 更新哈希对象
   │  └─ hash.update(chunk)
   ├─ 检查是否到达文件末尾
   │  └─ 未到达 → 继续读取
   └─ 到达 → 退出循环
   ↓
关闭文件
   ↓
获取最终哈希值
   ├─ 转换为十六进制字符串
   └─ 生成 64 字符字符串
   ↓
存储哈希值
   ├─ 保存到 media_files 表
   │  └─ sha256_hash 字段
   └─ 创建唯一索引
      └─ 优化查询性能
   ↓
返回哈希值
   └─ 用于后续文件判定
```

### 2.7 基于哈希值的已扫描文件判定流程

```
扫描器处理文件
   ↓
【计算当前文件哈希值】
   ↓
调用哈希计算函数
   ├─ calculate_file_hash(file_path)
   └─ 获取当前文件的 SHA-256 哈希值
   ↓
检查跳过策略
   ↓
根据 skip_strategy 判定
   ↓
【策略1: keyword】
   ↓
仅检查关键词库
   ├─ 查询关键词库
   │  └─ 匹配文件名
   ├─ 匹配关键词库 → 跳过扫描
   └─ 不匹配 → 继续扫描
   ↓
【策略2: keyword_or_scanned】
   ↓
先检查关键词库
   ├─ 查询关键词库
   ├─ 匹配关键词库 → 跳过扫描
   └─ 不匹配 → 继续检查
   ↓
检查是否已扫描（通过哈希值）
   ↓
查询数据库
   ├─ 查询 media_files 表
   │  └─ WHERE sha256_hash = ?
   │
   ├─ 找到相同哈希值
   │  ├─ 哈希值相同 → 文件已扫描
   │  ├─ 比较文件路径
   │  │  ├─ 路径相同 → 跳过扫描
   │  │  └─ 路径不同 → 重复文件，跳过扫描
   │  └─ 记录为 skipped
   │
   └─ 未找到相同哈希值
      └─ 文件未扫描 → 继续扫描
   ↓
【策略3: none】
   ↓
不执行任何跳过检查
   └─ 直接扫描文件
```

### 2.8 扫描任务执行流程

```
扫描器启动
   ↓
初始化扫描器
   ├─ task_id = 任务ID
   ├─ batch_id = 批次ID
   ├─ strategy = 扫描策略
   │  ├─ scan_type
   │  ├─ recursive
   │  ├─ skip_strategy
   │  └─ scan_subdirectories
   └─ progress_reporter = 进度报告器
   ↓
【遍历目录】
   ↓
根据 recursive 配置遍历
   ├─ recursive = true
   │  └─ 使用 rglob('*') 递归遍历
   │
   └─ recursive = false
      └─ 使用 glob('*') 非递归遍历
   ↓
【处理每个文件】
   ↓
获取文件列表
   ├─ 统计总文件数 (total_files)
   ├─ 初始化已扫描集合 (scanned_files = [])
   └─ 初始化统计计数
      ├─ new_files = []
      ├─ updated_files = []
      ├─ skipped_files = 0
      └─ failed_files = []
   ↓
对每个文件执行处理循环
   ↓
【文件处理循环】
   ↓
检查停止标志
   ├─ 已停止 → 退出循环
   └─ 未停止 → 继续
   ↓
文件类型双重判定
   ├─ 检查文件扩展名
   ├─ 检查编码格式
   ├─ 两者都符合 → 继续
   └─ 任一不符合 → 跳过
   ↓
【计算文件哈希值】
   ↓
计算 SHA-256 哈希
   ├─ 分块读取文件
   ├─ 更新哈希对象
   └─ 生成哈希字符串
   ↓
【跳过策略检查】
   ↓
根据 skip_strategy 检查
   ├─ skip_strategy = "keyword"
   │  └─ 检查是否为关键词库文件
   │     ├─ 文件名匹配关键词库 → 跳过
   │     └─ 不匹配 → 继续
   │
   ├─ skip_strategy = "keyword_or_scanned"
   │  ├─ 检查是否为关键词库文件
   │  │  ├─ 是 → 跳过
   │  │  └─ 否 → 继续检查
   │  └─ 检查文件是否已扫描（通过哈希值）
   │     ├─ 查询 media_files 表
   │     │  └─ WHERE sha256_hash = ?
   │     │
   │     ├─ 找到相同哈希值 → 跳过
   │     │  └─ skipped_files += 1
   │     │
   │     └─ 未找到 → 继续
   │
   └─ skip_strategy = "none"
      └─ 不跳过任何文件
   ↓
【创建/更新文件任务】
   ↓
创建 file_tasks 记录
   ├─ media_file_id
   ├─ scan_batch_id = batch_id
   ├─ file_name
   ├─ file_path
   ├─ status = "scanning"
   ├─ scan_started_at = 当前时间
   └─ scan_progress = 0
   ↓
【提取文件元数据】
   ↓
使用 pymediainfo 提取
   ├─ 文件基本信息
   │  ├─ file_size
   │  ├─ file_name
   │  ├─ file_extension
   │  └─ modify_time
   ├─ 媒体元数据
   │  ├─ duration
   │  ├─ width, height
   │  ├─ video_codec
   │  ├─ audio_codec
   │  ├─ video_tracks
   │  ├─ audio_tracks
   │  └─ subtitle_tracks
   └─ 计算文件哈希
      └─ sha256_hash
   ↓
【数据库操作】
   ↓
检查文件是否已存在（通过哈希值）
   ├─ 查询 media_files 表
   │  └─ WHERE sha256_hash = ?
   │
   ├─ 文件不存在
   │  ├─ 创建新 media_file 记录
   │  ├─ 插入所有元数据
   │  ├─ 插入哈希值
   │  ├─ new_files.append(file_data)
   │  └─ file_task.status = "scanned"
   │
   └─ 文件已存在
      ├─ 根据 scan_type 处理
      │
      ├─ scan_type = "full" (全量扫描)
      │  ├─ 强制更新 media_file
      │  ├─ 更新所有元数据
      │  ├─ 更新哈希值
      │  ├─ updated_at = 当前时间
      │  ├─ updated_files.append(file_data)
      │  └─ file_task.status = "scanned"
      │
      ├─ scan_type = "incremental" (增量扫描)
      │  ├─ 比较哈希值
      │  │  ├─ 哈希值相同 → 跳过
      │  │  │  ├─ skipped_files += 1
      │  │  │  └─ file_task.status = "skipped"
      │  │  └─ 哈希值不同 → 更新
      │  │     ├─ 更新 media_file
      │  │     ├─ 更新哈希值
      │  │     ├─ updated_files.append(file_data)
      │  │     └─ file_task.status = "scanned"
      │
      └─ scan_type = "custom" (自定义扫描)
         ├─ 强制更新 media_file
         ├─ 更新所有元数据
         ├─ 更新哈希值
         ├─ updated_files.append(file_data)
         └─ file_task.status = "scanned"
   ↓
更新 file_tasks 记录
   ├─ scan_completed_at = 当前时间
   ├─ scan_progress = 100
   ├─ video_tracks
   ├─ audio_tracks
   ├─ subtitle_tracks
   ├─ video_codec
   ├─ has_external_subtitle
   ├─ external_subtitle_name
   └─ scan_result = "success"
   ↓
【进度更新】
   ↓
检查防抖时间
   ├─ 距上次更新 < debounce_time
   │  └─ 跳过更新
   └─ 距上次更新 >= debounce_time
      └─ 更新进度
   ↓
构建进度数据
   ├─ task_id
   ├─ batch_id
   ├─ status = "running"
   ├─ total_files
   ├─ scanned_files = len(scanned_files)
   ├─ new_files = len(new_files)
   ├─ updated_files = len(updated_files)
   ├─ skipped_files = skipped_files
   ├─ failed_files = len(failed_files)
   ├─ progress = (scanned_files / total_files) * 100
   └─ current_file = 当前文件路径
   ↓
更新 scan_progress 表
   ├─ 更新进度记录
   └─ last_updated_at = 当前时间
   ↓
通过WebSocket推送
   └─ manager.send_progress(task_id, progress_data)
   ↓
【继续下一个文件】
   ↓
返回循环开始
   ↓
【扫描完成】
   ↓
所有文件处理完毕或被停止
   ↓
计算扫描耗时
   └─ duration = now - started_at
   ↓
确定最终状态
   ├─ 正常完成 → status = "completed"
   └─ 被停止 → status = "stopped"
   ↓
更新 scan_history 表
   ├─ total_files = len(scanned_files)
   ├─ new_files = len(new_files)
   ├─ updated_files = len(updated_files)
   ├─ skipped_files = skipped_files
   ├─ failed_files = len(failed_files)
   ├─ duration_seconds = duration
   ├─ completed_at = 当前时间
   └─ status = final_status
   ↓
更新 scan_progress 表
   ├─ status = final_status
   ├─ progress = 100
   ├─ completed_at = 当前时间
   └─ last_updated_at = 当前时间
   ↓
发送最终进度
   └─ WebSocket推送最终状态
   ↓
通知扫描队列管理器
   └─ on_task_completed(task_id, disk_letter)
   ↓
任务完成
```

### 2.9 扫描队列管理流程

```
【添加扫描任务】
   ↓
添加任务到队列
   ├─ 识别磁盘盘符
   │  ├─ Windows: C:\path → C
   │  └─ Linux: /mnt/disk1/path → /mnt/disk1
   │
   ├─ 检查磁盘队列是否存在
   │  └─ 不存在 → 创建队列
   │     └─ disk_queues[disk_letter] = []
   │
   ├─ 添加任务到队列
   │  └─ disk_queues[disk_letter].append(task_config)
   │
   └─ 尝试启动任务
      └─ _try_start_task(disk_letter)
   ↓
【尝试启动任务】
   ↓
检查活跃任务
   ├─ disk_letter in active_tasks
   │  └─ 该磁盘已有任务在执行
   │     └─ 返回，不启动新任务
   │
   └─ disk_letter not in active_tasks
      └─ 该磁盘无活跃任务
         ↓
检查队列是否为空
   ├─ 队列为空
   │  └─ 返回，无任务可启动
   │
   └─ 队列不为空
      └─ 有待执行任务
         ↓
从队列取出任务
   └─ task_config = disk_queues[disk_letter].pop(0)
   ↓
记录活跃任务
   └─ active_tasks[disk_letter] = task_config['task_id']
   ↓
启动扫描任务
   ├─ 创建扫描器实例
   │  └─ scanner = FileScanner(**task_config)
   │
   └─ 后台执行扫描
      └─ asyncio.create_task(scanner.scan_directory(...))
   ↓
【任务完成回调】
   ↓
任务完成或被停止
   ↓
清理活跃任务
   ├─ 检查任务ID是否匹配
   │  └─ active_tasks[disk_letter] == task_id
   │     └─ 是 → 从活跃任务中移除
   │        └─ del active_tasks[disk_letter]
   │
   └─ 尝试启动下一个任务
      └─ _try_start_task(disk_letter)
   ↓
【多磁盘并行执行】
   ↓
不同磁盘的任务可以并行执行
   ├─ C盘队列: 任务1 执行中
   ├─ D盘队列: 任务2 执行中
   └─ E盘队列: 任务3 执行中
   ↓
同一磁盘的任务串行执行
   └─ C盘队列:
      ├─ 任务1 执行中
      ├─ 任务2 等待中
      └─ 任务3 等待中
```

### 2.15 扫描进度实时推送流程

```
【扫描器端】
   ↓
扫描器处理文件
   ↓
文件处理完成
   ↓
构建进度数据
   ├─ task_id
   ├─ batch_id
   ├─ status
   ├─ total_files
   ├─ scanned_files
   ├─ new_files
   ├─ updated_files
   ├─ skipped_files
   ├─ failed_files
   ├─ progress
   └─ current_file
   ↓
调用进度报告器
   └─ progress_reporter.report_progress(progress_data)
   ↓
进度报告器检查防抖时间
   ├─ 当前时间 - 上次更新时间 >= debounce_time
   │  └─ 立即发送
   │
   └─ 当前时间 - 上次更新时间 < debounce_time
      └─ 等待防抖时间
         └─ 设置定时器
            └─ 防抖时间到后发送
   ↓
通过WebSocket发送
   └─ await manager.send_progress(task_id, progress_data)
      ├─ 获取活跃连接
      │  └─ active_connections.get(task_id, [])
      │
      ├─ 遍历所有连接
      │  └─ for connection in connections:
      │     └─ await connection.send_json(progress_data)
      │
      └─ 处理发送失败
         ├─ 捕获异常

   ↓
【前端端】
   ↓
WebSocket接收消息
   ↓
触发 onmessage 事件
   ↓

   ↓

   └─ scanProgress.set(taskId, progressData)

   ├─ 更新任务列表中的任务

   │  │
   │  └─ 更新任务属性
   │     ├─ task.progress = progress
   │     ├─ task.scannedFiles = scanned_files
   │     ├─ task.totalFiles = total_files
   │     ├─ task.newFiles = new_files
   │     ├─ task.updatedFiles = updated_files
   │     ├─ task.skippedFiles = skipped_files
   │     ├─ task.failedFiles = failed_files
   │     ├─ task.currentFile = current_file
   │     └─ task.status = status
   │
   ├─ 更新进度条
   │  └─ 进度条宽度 = progress + "%"
   │
   ├─ 更新文件列表
   │  ├─ 更新文件任务状态
   │  └─ 更新扫描进度
   │
   └─ 更新统计显示
      ├─ 总文件数
      ├─ 已扫描文件数
      ├─ 新增文件数
      ├─ 更新文件数
      └─ 跳过文件数
   ↓
【扫描完成】
   ↓
扫描器发送最终进度
   ├─ status = "completed" 或 "stopped"
   ├─ progress = 100

   ↓
前端接收最终进度
   ↓
更新任务状态为完成
   ├─ task.status = "completed" 或 "stopped"

   ↓
断开WebSocket连接
   ├─ 关闭进度监控
   └─ 释放资源
   ↓
刷新任务列表
   └─ 重新加载任务列表
```

## 六、关键特性说明

### 6.1 文件类型双重判定

**目的**: 确保只扫描真正的视频文件，避免误扫描

**实现**:
1. 第一层：文件扩展名判定
   - 检查文件扩展名是否在支持列表中
   - 支持22种常见视频容器格式

2. 第二层：文件编码格式判定
   - 使用 pymediainfo 提取视频编码信息
   - 检查编码格式是否在支持列表中
   - 支持10种主流视频编码格式

**优势**:
- 避免扫描伪装成视频的非视频文件
- 提高扫描准确性
- 减少无效扫描

### 6.2 基于SHA-256哈希值的文件判定

**目的**: 通过文件哈希值准确判定文件是否已扫描

**实现**:
1. 文件扫描完成后计算SHA-256哈希值
   - 分块读取文件（每次8MB）
   - 避免内存溢出
   - 生成64字符十六进制字符串

2. 存储哈希值到数据库
   - 存储在 media_files 表的 sha256_hash 字段
   - 创建唯一索引，优化查询性能

3. 增量扫描时的文件判定
   - 计算当前文件哈希值
   - 查询数据库中是否存在相同哈希值
   - 哈希值相同 → 文件已扫描，跳过
   - 哈希值不同 → 文件未扫描或已更新，继续扫描

**优势**:
- 基于文件内容而非文件名或时间戳
- 准确检测文件变更
- 避免重复扫描相同文件
- 支持文件去重

### 6.3 按磁盘控制的扫描队列

**目的**: 优化磁盘IO性能，特别是机械硬盘

**实现**:
- 同一磁盘：单线程扫描
- 不同磁盘：每个磁盘一个扫描线程
- 队列管理：每个磁盘独立队列

**优势**:
- 避免磁盘头频繁移动
- 提高机械硬盘性能
- 支持多磁盘并行扫描

### 6.4 可配置的防抖机制

**目的**: 避免频繁更新影响性能

**实现**:
- 扫描进度防抖：30-300秒
- 文件监控防抖：1-60秒
- 配置灵活，适应不同场景

**优势**:
- 减少数据库写入
- 降低网络传输
- 提高系统响应速度
